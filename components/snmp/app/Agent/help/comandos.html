4.- Comandos de Control
Son los comandos que permiten que el agente llegue a integrarse como un componente más de ISABEL. De todas formas si el agente se arranca de forma independiente, con el parámetro -shell, es capaz de recibir los comandos de control por la consola desde la que se ejecutó. El conjunto de comandos de control son:i

  • snmp_nop(): a este comando el programa responde con OK, si es que está lanzado. Este comando lo necesita ISABEL para comprobar si el agente está lanzado.
  • snmp_bye() y snmp_quit(): hacen que el agente finalice su ejecución.
  • snmp_gui(valor): permite lanzar y parar la interfaz gráfica del agente en cualquier momento. Los posible valores son:
    • snmp_gui(0): si estaba lanzada, la para. La próxima vez que se lance la interfaz habrán desaparecido todas las ventanas de gráficos que tuviéramos abiertas, así como las de ayuda o configuración. Al relanzar sólo estará la ventana principal del agente.
    • Snmp_gui(1): lanza la interfaz gráfica, si es que no estaba lanzada ya.
